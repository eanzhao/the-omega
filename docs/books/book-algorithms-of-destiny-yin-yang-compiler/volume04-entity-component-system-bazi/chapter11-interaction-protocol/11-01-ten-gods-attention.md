# 第十一章：交互协议：十神注意力机制

**Chapter 11: Interaction Protocol: Ten Gods Attention Mechanism**

## 11.1 注意力权重：十神作为接口协议

我们将八字看作一个Transformer模型，**十神（Ten Gods）** 就是各种类型的**注意力头（Attention Heads）**。
它们定义了“我”如何关注并处理外部信息。

1.  **印枭 (Resource)**: **Input Attention**。
    *   负责接收信息、学习、充能。
    *   印旺的人，模型的输入带宽大，缓存（记忆）深。
2.  **食伤 (Output)**: **Output Attention**。
    *   负责表达、创造、宣泄。
    *   食伤旺的人，模型的生成能力强，Token输出速度快。
3.  **官杀 (Control)**: **Constraint / Loss Function**。
    *   负责约束、规则、压力。
    *   官杀旺的人，模型的边界条件（Boundary Conditions）严格，自我纠错能力强。
4.  **财星 (Target)**: **Object Attention**。
    *   负责控制对象、资源、目标。
    *   财旺的人，模型的目标导向性（Goal-Oriented）强，善于处理实际数据。
5.  **比劫 (Peer)**: **Self-Attention**。
    *   负责自我强化、同伴连接。
    *   比劫旺的人，模型的自我权重高，意志力强，但也容易过拟合（固执）。

## 11.2 六亲网络：社会关系图谱

通过这十种注意力机制的组合，八字构建了一个复杂的**社会关系图谱（Social Graph）**。
命运的吉凶，往往不取决于单个节点的能量强弱，而取决于**整个网络的拓扑结构**是否通畅。

*   **通关（Routing）**：当“官杀”（压力）太重克身时，如果有一个“印”（输入）作为路由器，将压力转化为动力（官生印，印生身），这叫“杀印相生”。这是系统设计中的**负载均衡（Load Balancing）**。
