# 第八章：交互协议：十神注意力机制

**Chapter 8: Interaction Protocol: Ten Gods Attention Mechanism**

## 8.1 注意力权重：十神作为自我与世界交互的接口

我们将八字系统重构为一个 **Transformer 模型**。
**十神 (Ten Gods)** 不是迷信的神灵，而是 10 个**多头注意力 (Multi-Head Attention)** 模块。
它们定义了 Agent 如何分配有限的计算资源（注意力）。

### 8.1.1 输入/输出层
1.  **印枭 (Resource)**: **Input Head**。
    *   功能：数据摄入、模型训练、权重更新。
    *   物理含义：负熵流的输入通道。
2.  **食伤 (Output)**: **Output Head**。
    *   功能：数据生成、表达、行动。
    *   物理含义：做功 (Work)，消耗内部能量以改变外部环境。

### 8.1.2 约束/目标层
3.  **官杀 (Control)**: **Loss Function (损失函数)**。
    *   功能：计算误差、施加惩罚、正则化 (Regularization)。
    *   物理含义：外部环境对系统的约束力/边界条件。
4.  **财星 (Target)**: **Objective Function (目标函数)**。
    *   功能：定义优化的目标（获取资源、控制对象）。
    *   物理含义：系统试图最小化自由能的对象。

### 8.1.3 自我层
5.  **比劫 (Peer)**: **Residual Connection (残差连接)**。
    *   功能：保持身份的同一性，防止梯度消失。
    *   物理含义：系统的惯性质量 (Inertial Mass)。

## 8.2 六亲网络：社会关系图谱

八字不仅模拟个体，还模拟**多体相互作用 (Many-Body Interaction)**。
通过十神，系统构建了一个**动态社交网络图 (Dynamic Social Graph)**。
*   **生**: 节点间的正权重连接。
*   **克**: 节点间的负权重连接。
命运的吉凶，取决于这个网络在特定时间步（流年）下的**拓扑稳定性**。
如果某个关键节点（用神）被移除（冲克），可能导致整个网络发生**级联失效 (Cascading Failure)**。
