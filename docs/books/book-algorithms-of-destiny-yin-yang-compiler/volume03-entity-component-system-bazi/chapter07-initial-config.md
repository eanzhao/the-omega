# 第七章：初始配置：四柱架构解析

**Chapter 7: Initial Configuration: Four Pillars Architecture**

## 7.1 用户实例：日元作为相对论参考系的原点确立

### 7.1.1 八字即模拟参数
在计算宇宙中，每个生命体都是一个独立运行的 **Agent**。
八字（四柱）是该 Agent 出厂时的**初始配置文件 (config.json)**。
它定义了 Agent 的：
*   **硬件规格** (五行强弱)
*   **接口类型** (十神结构)
*   **系统时钟** (大运流年)

### 7.1.2 观察者参考系 (Observer Reference Frame)
为什么以**日干**为我？
在相对论物理中，物理定律的形式取决于参考系。
对于一个个体而言，最稳定的惯性参考系不是地球公转（年），也不是地球自转（时），而是**昼夜交替的节奏**（日）。
确立日元，就是确立了希尔伯特空间中的**基态矢量 $|\psi_{self}\rangle$**。
所有其他的干支（年、月、时），都是相对于这个基态矢量的**算符 (Operators)**，对“我”施加影响。

## 7.2 环境参数：月令权重与系统初始温度

### 7.2.1 月令：热力学背景温度
月令（出生月份）不仅仅是一个五行参数，它是系统的**背景温度 (Background Temperature) $T$**。
*   生于午月（夏）：$T$ 极高。系统处于高熵状态，粒子运动剧烈。需要“水”作为**冷却剂 (Coolant)** 来降低熵，维持结构稳定。
*   生于子月（冬）：$T$ 极低。系统处于冻结状态，活性低。需要“火”作为**激活剂 (Activator)** 来提供活化能。

**调候 (Climate Adjustment)** 的本质，就是**热力学平衡 (Thermodynamic Equilibrium)** 的调节。
脱离了温度谈五行生克，就像是在不知道温度的情况下计算化学反应速率，是毫无意义的。
